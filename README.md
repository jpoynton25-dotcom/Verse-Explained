# Verse Explained

Production-ready, SEO-first Next.js website for **Verse Explained** (`VerseExplained.com`) with a content-driven architecture focused on ranking for Bible verse meaning, daily verse, verses by need, Bible terms, and Bible people/places.

## Purpose
Verse Explained helps readers understand Bible verses in plain English with clear context, practical meaning, and non-denominational tone.

## Tech Stack
- Next.js App Router
- TypeScript
- Tailwind CSS
- Static generation for content pages via `generateStaticParams`
- Server Components by default

## Project Structure
- `app/`: routes, layout, hub pages, dynamic content route, sitemap, robots
- `components/`: reusable layout, content, SEO, navigation, and UI components
- `content/`: structured content collections by pillar
- `lib/`: site config, SEO metadata helpers, schema builders, content aggregation
- `public/`: OG image asset
- `BACKLINK_STRATEGY.md`: realistic off-page and authority plan

## Content Model
Each content page is data-driven from `content/*.ts` and rendered by one scalable route: `app/[slug]/page.tsx`.

## SEO Architecture
- Centralized site config in `lib/siteConfig.ts`
- Centralized metadata + canonical helper in `lib/seo.ts`
- Schema helpers in `lib/schema.ts`:
  - Organization
  - Website (+ SearchAction)
  - BreadcrumbList
  - Article
  - FAQPage
- Dynamic metadata per content page in `app/[slug]/page.tsx`
- Canonical URLs on all pages via metadata alternates
- Open Graph and Twitter card metadata on all pages via helper
- `app/sitemap.ts` and `app/robots.ts`
- Internal linking:
  - Home -> hubs
  - Hubs -> child pages
  - Child pages -> hubs + related pages
  - Contextual links between pillars

## How to Add New Pages
1. Add a new object to the appropriate file in `content/`.
2. Ensure:
   - unique `slug`
   - unique `seoTitle`
   - unique `metaDescription`
   - clear `h1` and intro aligned with intent
   - related slugs in `relatedSlugs`
   - FAQ entries
3. The page is auto-generated by `app/[slug]/page.tsx` and auto-included in sitemap.
4. Optionally feature it on homepage or relevant hub.

## Content QA Checklist (Future Pages)
Use this checklist before publishing any new page:
1. Intro quality:
   - The intro is clear, natural, and aligned to the main keyword intent.
2. Heading structure:
   - Headings follow the page template and include practical sections (meaning, context, takeaway, FAQ).
3. Internal links:
   - Page links to relevant hubs and related pages.
   - At least 2 existing pages link back to the new page.
4. FAQ quality:
   - 3-5 real user questions with concise, practical answers.
5. Schema present:
   - Breadcrumb schema and article/webpage schema are generated.
   - FAQ schema appears only when visible FAQ content exists.
6. No placeholder copy:
   - No TODO/TBD/placeholder text or unfinished sections.
7. No repeated filler language:
   - Remove generic AI-sounding phrases and tighten long sentences.

## Deployment
1. Push to Git provider.
2. Deploy on Vercel (recommended for App Router and static generation).
3. Set production domain to `www.verseexplained.com`.
4. Confirm:
   - `/robots.txt`
   - `/sitemap.xml`
   - canonical tags
   - OG/Twitter previews

## Launch Checklist
1. Connect domain:
   - Add `www.verseexplained.com` and apex domain in hosting provider.
   - Confirm SSL and redirect rules (`http` -> `https`, apex -> `www` or preferred canonical host).
2. Production environment setup:
   - Set production environment variables (if analytics/email provider is added).
   - Verify build command and Node version in hosting settings.
3. Search Console:
   - Verify property in Google Search Console.
   - Submit `https://www.verseexplained.com/sitemap.xml`.
4. Analytics:
   - Add analytics provider (GA4 or privacy-focused alternative).
   - Validate pageview tracking on hubs and content pages.
5. Final QA pass:
   - Check mobile layout for homepage and all hubs.
   - Validate canonical/OG metadata on top pages.
   - Spot-check internal links and schema output.
   - Confirm `robots.txt` and `sitemap.xml` resolve in production.
